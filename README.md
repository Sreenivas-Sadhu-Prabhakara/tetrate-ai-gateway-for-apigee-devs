# Tetrate AI Gateway for Apigee & Java Developers

A from-the-basics learning site for the **Tetrate / Envoy AI Gateway**, aimed at
**Java / Spring developers who already know Apigee**. Every concept is taught with a
**double anchor** — bridged from the Apigee object you'd reach for *and* from the
Spring code you write today — then we show what's genuinely new because the payload
is now **tokens, prompts, and tool calls**. Hands-on labs throughout, an inline
**"Watch out"** pitfall in every section, and a cool **Mayari** (moon) dark theme.

**Live site:** https://sreenivas-sadhu-prabhakara.github.io/tetrate-ai-gateway-for-apigee-devs/

It's the sibling of the companion course **Apigee X for Spring Boot Developers**
(the warm *Apolaki* sun theme); this one is the cool *Mayari* moon.

## Course shape

**29 atomic, cumulative sessions across 7 parts.** Each session is self-contained —
one objective, an explicit *builds-on* link, **From Apigee** and **From Java
microservices** bridges (and where the analogy breaks), a hands-on lab, *verify it*,
common failure modes, and a stretch goal.

1. **Why an AI gateway exists** — what it is, why direct LLM calls stop scaling, Tetrate's lay of the land (TARS, Agent Router Enterprise, Agent Operations Director, Envoy AI Gateway).
2. **The unified multi-provider gateway** — one OpenAI-compatible API; `AIServiceBackend`, `BackendSecurityPolicy`, `AIGatewayRoute`; routing, fallback, embeddings.
3. **Governance: tokens, cost & quotas** — token-based rate limiting, budgets & cost attribution, sanctioned-model tiers, caching.
4. **Security & guardrails** — caller auth, prompt-injection defense, PII redaction, output moderation, securing the upstream.
5. **The agentic turn: MCP & tools** — from chat to agents, `MCPRoute`, tool routing/multiplexing, OAuth on tools, agent identity & budgets.
6. **Observability & operations** — token/latency/OpenInference tracing, debugging, Kubernetes / Gateway API / `aigw` CLI, production readiness.
7. **Capstone** — assemble a governed, observable, multi-provider agent platform on Tetrate.

## Build

```bash
pip install -r requirements.txt
python3 build.py            # renders content/ → docs/
python3 -m http.server -d docs 8000   # http://localhost:8000  (widgets need HTTP)
```

The site is **pre-rendered** — `docs/` is committed and served by GitHub Pages
(`main` → `/docs`) with `.nojekyll`. Regenerate locally with `python3 build.py`.

## Authoring a session

Each `content/session-NN.md` starts with `# <code> — <title>` and uses the Mayari
callout components (Markdown admonitions), rendered by `build.py`:

```markdown
!!! bottomline "Bottom line"
!!! apigee "From Apigee"            (the Apigee anchor)
!!! java "From Java microservices"  (the Spring/Java anchor)
!!! breaks "Where the analogy breaks"
!!! pitfall "Watch out"             (inline gotcha, in every section)
!!! verify / !!! failure / !!! stretch
```

The code-bearing **Lab** uses `<div class="lab" markdown="1">` (not an admonition),
because python-markdown's fenced code does not render inside `!!!` blocks. Keep all
fenced code at column 0 — in a `##` section or inside the lab div. Crumbs, the
builds-on band, the hero (on part openers), the "mark complete" control, and prev/next
are generated by `build.py` from `curriculum.json`.

## Accuracy note

Labs are grounded in the public Tetrate and **Envoy AI Gateway** documentation, but
the AI-gateway space moves fast and CRD API versions/fields evolve per release. Each
YAML block notes "verify field names against the docs for your release." Always
validate against the current official docs before production use.
